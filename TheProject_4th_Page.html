    The variable add is assigned to the return value of a self-invoking function.

    The self-invoking function only runs once. It sets the counter to zero (0), and returns a function expression.

    This way add becomes a function. It can access the counter in the parent scope.

    This is called a JavaScript closure. It makes it possible for a function to have "private" variables.

    The counter is protected by the scope of the anonymous function, and can only be changed using the add function.

    A closure is a function having access to the parent scope, even after the parent function has closed.

    The element node "p" or other do not contain text.

    It contains a text node with the value.

    The value of the text node can be accessed by the node's innerHTML property:

    myTitle = document.getElementById("demo").innerHTML;
    Accessing the innerHTML property is the same as accessing the nodeValue of the first child:

    myTitle = document.getElementById("demo").firstChild.nodeValue;
    Accessing the first child can also be done like this:

    myTitle = document.getElementById("demo").childNodes[0].nodeValue;

The information from the navigator object can often be misleading.

The navigator object should not be used to detect browser versions because:

Different browsers can use the same name
The navigator data can be changed by the browser owner
Some browsers misidentify themselves to bypass site tests
Browsers cannot report new operating systems, released later than the browser

The validity property of an input element contains a number of properties related to the validity of data:

Property	Description
customError	Set to true, if a custom validity message is set.
patternMismatch	Set to true, if an element's value does not match its pattern attribute.
rangeOverflow	Set to true, if an element's value is greater than its max attribute.
rangeUnderflow	Set to true, if an element's value is less than its min attribute.
stepMismatch	Set to true, if an element's value is invalid per its step attribute.
tooLong	        Set to true, if an element's value exceeds its maxLength attribute.
typeMismatch	Set to true, if an element's value is invalid per its type attribute.
valueMissing	Set to true, if an element (with a required attribute) has no value.
valid	        Set to true, if an element's value is valid.
